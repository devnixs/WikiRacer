# Story 013: Match Screen Layout And Race Status

## Goal

Build the core gameplay screen for desktop and mobile.

## Scope

- Render the article content in the main reading area.
- Show the table of contents in a persistent left panel on desktop.
- Show a collapsible table of contents on mobile.
- Show race status with timer, player pages, hop counts, and player state.

## Acceptance Criteria

- Desktop layout matches the specified article, TOC, and race-status structure.
- Mobile layout keeps the article readable and TOC collapsible.
- The timer remains visible during play.
- Player pages, hop counts, and statuses update in real time.
